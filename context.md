## 🏛️ 3. Модуль «Опись основных средств (ОС и НМА)»

Модуль предназначен для физической и финансовой инвентаризации оборудования, кухонной техники, мебели и IT-устройств ресторана.

### Архитектура модуля:
* **База данных (`Supabase`):** Таблица `fixed_assets`
  * Поля: `id`, `inv_number`, `name`, `category` (`equipment`, `furniture`, `it_tech`, `software_nma`, `transport`, `real_estate`, `other`), `initial_cost`, `salvage_value`, `useful_life_months`, `commissioning_date`, `depreciation_method`, `accumulated_depreciation`, `book_value`, `status` (`in_use`, `in_stock`, `repair`, `written_off`, `sold`), `responsible_person` (МОЛ), `location`, `serial_number`, `iiko_equipment_id`, `qr_code_data`.
* **API Эндпоинты:**
  * `/api/iiko/assets` (GET, POST, PUT, DELETE) — CRUD-операции, автоматический перерасчет амортизации, фильтрация по статусу, поиску и локации.
  * `/api/iiko/assets/sync` (POST) — импорт и синхронизация оборудования из iiko Office / iiko Web.
* **UI Компонент:** `FixedAssetsView` внутри `components/LocmacoApp.jsx`
  * KPI карточки: Балансовая стоимость, Первоначальная стоимость, Накопленная амортизация, Амортизация за текущий месяц.
  * Печать стикеров: Генератор QR-кодов и инвентарных номеров для наклеивания на физическое оборудование.

## 🧾 4. Бухгалтерский и налоговый контекст (1С и iiko)

### 4.1. Специфика закупа (Безнал vs Базар):
1. **Официальный закуп (Безнал / КПК / Расчетный счет):**
   * Поставщики с формами MCHJ, ООО, XK, AJ (например, *FAR TRADE EXPERT MCHJ*).
   * Имеется официальный счет-фактура / КП. Дает 100% зачет НДС и принимается в 1С.
   * Примеры товаров: Мука (*Мука*, *Мука Mama Mia* — 100% безнал), упаковка, фирменные соусы, напитки.
2. **Наличный закуп (Базар / Снабженец):**
   * Закупки производит снабженец на базаре за наличные.
   * Первичных официальных документов нет.
   * Примеры товаров: Свежие ягоды (клубника, банан), зелень, свежие овощи (доля безнала ~1%).

### 4.2. Правило списания для 1С (Коэффициент списания):
* ❌ **НЕЛЬЗЯ использовать единый плоский процент (например, 35-40% на всё):**
  * По товарам со 100% безналом (мука) возникнет гигантский виртуальный пересорт в 1С (приход 100%, списание 35%).
  * По товарам с базара (бананы) 1С уйдет в **красные минусовые остатки на счетах ТМЦ (2910/1010)**, так как безналичного прихода не было.
* ✅ **ПРАВИЛЬНЫЙ ПОДХОД:** Применять дифференцированные коэффициенты по группам номенклатур (на основе скрипта `calculate_all_ingredients_ratio.mjs`):
  * Группа 100% безнал (мука, упаковка): Списание 100% от расхода iiko (коэфф `1.0`).
  * Группа смешанного закупа (мясо, молочка): Списание по расчитанному коэффициенту (напр. `0.65`–`0.75`).
  * Группа чисто наличного закупа (базар): Списание только в пределах имеющихся безналичных накладных / авансовых отчетов.

## ⚙️ 5. Специфика работы с iiko API

* **Переменные окружения (`.env`):**
  * `IIKO_SERVER`: `https://the-lokmako.iiko.it`
  * `IIKO_LOGIN`: `asil`
  * `IIKO_PASSWORD`: `asil12` (Авторизация через sha1 хэш: `/resto/api/auth?login=...&pass=sha1(...)`)
  * `IIKO_WEB_URL`: `https://the-lokmako.iikoweb.ru`
* **Особенность XML формата `incomingInvoice`:**
  * Запрос `/resto/api/documents/export/incomingInvoice` с заголовком `Accept: application/json` возвращает массив пустых объектов `[{}, {}]`.
  * **ОБЯЗАТЕЛЬНО** передавать `Accept: application/xml` и парсить ответ с помощью XML-парсера или регулярных выражений.
